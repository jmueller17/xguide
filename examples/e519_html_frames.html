e519 - HTML - frames

1.
 Using frames

Frames give you the possibility to divide the browser window into two, three, ... independent zones. You can refresh the content of one frame without touching the other where each frame contains a normal, complete, independent HTML file. It's like you put another browser window inside the current window.

With the bandwidth available today frames have somehow ceased to be in fashion. The advantage of frames is, that items which stay the same for your whole site such as the navigation menu don't have to be reloaded each time. You just place them in one frame and load the new page into the neighour frame. This bandwidth argument, however, has lost its power, since images stay in the browsers cache anyway and don't really occupie bandwidth once they have downloaded.

Anyway, some time you might want to organize your site with frames. So let's see how it is done. The trick is to think in three HTML files from the beginning. You have file ONE which provides the base frame structure. This file contains actually the frame tag. In this file you decide how you want to organize your frames. Then you have the actual content which are just ordinary independent HTML files which you could load in a normal browser window.

 

2.
 Code - I - two vertical frames

<html>
 <head>
  <title>e519a - HTML - Frameset</title>
 </head>

 <frameset cols="150,*" rows="*" frameborder="NO" border="0" framespacing="0" >
   <frame name="leftFrame" scrolling="NO" noresize src="e519a_html_framesLeft.html">
   <frame name="mainFrame" src="e519a_html_framesRight.html">
 </frameset>
 <noframes>
   <body bgcolor="#FFFFFF" text="#000000">
      Alternative message: Your browser cannot display frames!!
   </body>
 </noframes>
</html>

 
 

The first thing to note is that there is no <body> tag. Right after the closing head-tag we find the <frameset>. This means that this HTML file actually has no content itself but rather is used to establish the structure for two other pages. Note that the <body> tag is inside the <noframes> tag which means, that it will only be displayed if we have a browser which doesn't understand frames.

The next thing to note is the distribution of the frames. Inside the <frameset> tag we decide how to split the browser window. In this case two columns, where the first one has a width of "30%" of the browser window and the second will adapt to the rest. You could also use a pixel value instead. And we define to have just one row. All in all, the browser window will split in two vertically.

Then, after we defined how many frames we want, it's time to reference those. This happens inside the <frameset> These are the pages that will be displayed. The first on the left side, the second frame "mainFrame" on the right. The "src" attribute should be pretty familiar and specifies the actual path.

Take also note of the name attributes. For example, if you want to load new content by clicking on a link in the left frame to be shown in the right frame you have to use the target attribute the following way:

<a href="myPage.html" target="mainFrame">go to my page</a>

3.
 Code - II - three horizontal frames

<html>
 <head>
  <title>e519b - HTML - Frameset</title>
 </head>

 <frameset cols="*" rows="100,*,100" frameborder="NO" border="0" framespacing="0" >
   <frame name="topFrame" scrolling="NO" noresize src="e519b_html_framesTop.html">
   <frame name="middleFrame" src="e519b_html_framesMiddle.html">
   <frame name="bottomFrame" src="e519b_html_framesBottom.html">
 </frameset>
 <noframes>
   <body bgcolor="#FFFFFF" text="#000000">
      Alternative message: Your browser cannot display frames!!
   </body>
 </noframes>
</html>

 
 

This is basically the same example like the one above. We just changed the direction: now we have three horizontal frames where by the middle one adapts when resized. The top and bottom one has a fixed size.

You can download those three files as a zip file here or browse directly to them here.
