e518 - HTML - Form Elements

1.
 Forms

In order to insert a form into your HTML document use the following code:

<form name="testForm" action="http://www.myserver.com/receiveForm.php">

</form>

The example defines a form which has a name "testForm" and will send the data once submitted to the URL specified in the action parameter. In order to retrieve the values defined, you have to have a working script like the "receiveForm.php". This will be treated in the final sessions of class on PHP.

All form elements that follow below have to be enclosed by this form tags. This allows you to define several forms in one HTML page and nevertheless define their values perfectly.

 


2.
 Overview Form Elements

Normal text field. The name attribute will identify the value retrieved.

<input type="text" name="surname" />

 
 

The password field is actually an attribute of a normal text field. Please be aware that this does not guarantee a safe transmission over the web. This dots are only there to not show what you type.

<input name="textfield2" type="password" value="hello" />

 
 

Textarea for longer (email) texts for example.

<textarea name="textarea" cols="15" rows="5"></textarea>

3.
 Further tags
  <label for="surname">Surname:</label>
<input type="text" name="surname" id="surname" />

The <label> tag establish a logical connection between the descriptive text of a form element and the actual field. The connection - important for accessibility issues - is established between the "for" and the "id" attribute.

 

<fieldset>
   <legend>Adress</legend>
   <label for="surname">Surname:</label)
   <input type="text" name="surname" id="surname">
   ...
</fieldset>
<fieldset>
   <legend>Wishlist</legend>
   ...
</fieldset>

Forms can get quite long. With the <fieldset> tag you can establish groups of input fields. The <legend> is the title of a specific fieldset.
